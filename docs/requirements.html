1. サービス概要

コンセプト: 運送会社同士の助け合いプラットフォーム。帰り便（空車）にマッチングで荷物を載せ、空車率を下げる。荷主も参加可能。

🏢

ソロモード

SOLO

自社の配車状況を一元管理。車番・車種・運転手名・スケジュール（何時にどこからどこまで何を運ぶか）を入力。1つの地図で全トラックの予測位置を一覧表示。

🌐

マルチプラットフォームモード

MULTI

自社配車の一部工程（帰り便の空車区間等）を全体に公開。他社や荷主とマッチングし、チャットでやり取り。

具体例（神奈川県 / 本社: 横浜市）

トラック往路帰路帰り荷
① 2t横浜 → 川崎（9:00発 / 14:00着）川崎 → 横浜あり
② 4t横浜 → 厚木（9:00発 / 15:00着）厚木 → 横浜あり
③ 10t横浜 → 小田原（9:00発 / 15:00着）小田原 → 横浜なし → 募集!

→ トラック③の帰路が空車。この区間を外部公開してマッチング対象にする。

配車計画の単位

4. マルチプラットフォームモード（公開マッチング）

公開範囲（区間ごとに設定）

設定誰に見えるかユースケース
private自社のみ自社配車管理用
company_only運送会社ユーザーのみ運送会社同士のマッチング
public荷主含む全ユーザー荷主からの依頼も受付

マッチング対象になる条件

  • 区間の cargo_status が empty_seeking（空車・募集中）
  • 区間の visibility が public または company_only
  • 区間の status が scheduled（まだ配送開始前）

6. 予測位置情報

📍

GPSなし・出発/到着時刻ベースの位置予測

前提: GPSは運転手が嫌がるため使わない。出発時刻と到着時刻を入力し、その間の「大体の現在地」を予測表示する。あくまで目安。

  A地点（横浜）              C地点（予測位置）             B地点（小田原）
  9:00 出発                   12:00 ← いまこの辺?         15:00 到着
  ●━━━━━━━━━━━━━━━━━━━━●━━━━━━━━━━━━━━━━━━━━━━●
  ├── 経過 3時間 (50%) ──┤                              │
  │                        ├────── 残り 3時間 ──────────┤

  計算ロジック:
  1. Google Routes API で A→B のルート＋ポリラインを取得
  2. 出発時刻からの経過時間を計算
  3. 各ステップの duration を積算し、経過時間に対応する位置を線形補間
  4. その緯度経度にマーカーを表示

地図表示（ソロモード）

1つの地図に自社全トラック（例: ①②③）のルートを重ねて表示。各トラックの予測位置をリアルタイム風に表示。

精度向上オプション（GPSなし）

方法説明運転手の負担
ワンタップ報告「出発」「休憩開始/終了」「到着」ボタン最小限 ✅
遅延手動入力配車担当が「休憩30分」「渋滞20分遅れ」を入力なし（担当者が操作）
チェックポイントSA/IC名を選んで通過報告少し

※ traffic-aware polyline（追加課金）は使わない。標準のRoutes APIのみ使用。

7. チャット機能

マッチング承認後、自動でチャットルームが作成される。決済完了まではチャットのみ（電話番号は非公開）。

10. 優先度

優先度項目理由
🔴DB再設計（会社・配車計画・区間の階層構造）全機能の基盤。これがないと何も進まない
🔴ソロモード（配車管理＋地図表示）運送会社が最初に使う機能。これ単体で価値がある
🔴段階的情報開示マネタイズの要。直電回避の仕組み
🟡マルチプラットフォーム（公開マッチング）収益の柱。ソロモード完成後に着手
🟡チャット＋フィルタリングマッチング後のコミュニケーション
🟡Stripe決済（Payment Intent）課金フロー。マッチング完了の前提
🟢予測位置情報ソロモードの付加価値。なくても最低限動く
🟢評価・レビュー信頼構築。初期はなくてもOK
🔵サブスクリプションユーザー数が増えてから
🔵管理者ダッシュボード拡充運用フェーズで随時追加
